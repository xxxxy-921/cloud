## Context

当前聊天界面位于 `web/src/apps/ai/pages/chat/`，采用传统气泡对话设计：
- `message-bubble.tsx`: 用户消息右对齐气泡，AI 消息左对齐气泡
- `[sid].tsx`: 三栏布局（侧边栏 + 聊天区 + 记忆面板）
- 输入框在底部，使用单行 Textarea

参考现代 AI 产品（Claude.ai、ChatGPT、Perplexity）的设计演进，用户更期望文档流式的阅读体验。

## Goals / Non-Goals

**Goals:**
- 移除气泡视觉噪音，采用扁平化设计
- 最大化内容阅读区域（提升 max-width）
- 优化 Markdown 内容的排版层次
- 支持多行输入和自动增高
- 侧边栏可收起，释放空间

**Non-Goals:**
- 不改变任何 API 接口
- 不改变消息数据结构
- 不改变会话管理逻辑
- 不添加新的交互模式（如语音、图片上传）

## Decisions

### 1. 单栏全宽布局 vs 双栏
**选择**: 单栏全宽布局
**原因**: 参考 Claude 和最新 ChatGPT，所有内容左对齐，阅读流更自然。双栏左右分栏会造成视线跳跃。
**替代方案**: 保留左右分栏但去掉气泡——被认为仍有视线跳跃问题

### 2. 用户消息呈现方式
**选择**: 作为"引用"样式，左侧带边框，略小号字体
**原因**: 需要区分用户输入和 AI 回答，但不应使用气泡。
**替代方案**: 完全平铺仅用间距区分——难以快速定位对话起点

### 3. 输入框设计
**选择**: 底部固定悬浮卡片，支持多行自动增高
**原因**: 参考 Claude Composer，视觉上突出输入区域，同时不占用过多空间
**替代方案**: 底部工具栏式——过于紧凑，多行输入体验差

### 4. 侧边栏改造
**选择**: 改为可收起式（默认展开），提供 [≡] 按钮
**原因**: 保留现有功能，同时给用户选择权
**替代方案**: 顶部下拉式——会话多时难以浏览

### 5. 代码块样式
**选择**: 深色主题，带语言标签和悬浮复制按钮
**原因**: 代码是 AI 回答的重要部分，需要高对比度和便捷操作
**替代方案**: 浅色主题——与整体浅色背景对比度不足

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| 用户习惯原有气泡设计 | 保留清晰的视觉区分，过渡期内提供帮助提示 |
| 宽屏下长文本行过长 | max-width 限制在 65ch-75ch，保证可读性 |
| 移动端适配复杂度 | 先完成桌面版，移动端单独处理侧边栏 |
| 深色模式颜色一致性 | 使用 CSS 变量，确保所有颜色随主题切换 |
