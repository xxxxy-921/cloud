# 工单监控

工单监控用于跨用户观察 ITSM 工单状态，并在必要时手工推进流程。你可以从这里统一查看活跃工单、风险工单、SLA 风险、AI 异常，也可以切到“我的工单”“我的待办”“历史工单”处理与自己相关的任务。

## 一、这个模块解决什么问题

工单监控把“全局巡检”和“个人处理入口”放在同一个工单域里，既适合管理员排查卡单，也适合普通用户跟踪自己提交或待审批的工单。

当你需要定位工单卡在哪一步、确认谁在处理、查看时间线、审批节点或手工接管动作时，应该优先进入这里。

## 二、谁会在什么场景下使用

- ITSM 管理员在日常巡检时查看活跃量、卡单、风险和 AI 异常，并决定是否要人工干预。
- 服务负责人在工单详情中核对表单数据、时间线和当前步骤，判断流程有没有跑偏。
- 普通用户通过“我的工单”跟踪自己提交过的单据进展。
- 审批人通过“我的待办”和“历史工单”集中处理人工审批节点。

## 三、用户从哪个 Web 入口进入

- 左侧菜单：`ITSM > 工单监控`
- 工作台入口：`ITSM > 我的工单`、`ITSM > 我的待办`、`ITSM > 历史工单`
- 全局监控路由：`/itsm/tickets`
- 个人列表路由：`/itsm/tickets/mine`
- 审批列表路由：`/itsm/tickets/approvals/pending`、`/itsm/tickets/approvals/history`
- 详情入口：点击任一列表行进入 `/itsm/tickets/:id`

## 四、关键对象、状态与入口

- 全局监控首页会先展示活跃、卡单、风险、SLA 风险、AI 异常、今日完成等指标。
- 列表支持按关键字、风险等级、引擎类型、状态、优先级、所属服务等条件筛选。
- “我的工单”强调我提交过的单据；“我的待办”强调当前需要我审批的节点；“历史工单”强调我已处理过的节点。
- 工单详情至少会汇总表单数据、时间线、流程图、当前责任人、SLA 状态和下一步建议。
- 可操作入口包括 `指派`、`通过`、`驳回`、`取消工单`、`撤回工单`，不同入口受工单状态和权限限制。

## 五、常见任务流程

### 任务 1：从全局视图定位卡单并进入详情排查

路径：`ITSM > 工单监控`

步骤：
1. 先看顶部指标，确认当前是卡单、风险还是 SLA 风险突出。
2. 使用风险等级、状态、服务或优先级筛选工单列表。
3. 点击目标工单进入详情页。
4. 在详情页结合时间线、当前责任人、流程图和下一步建议判断卡点。

结果：
- 可以快速缩小问题范围，定位卡在哪个节点和谁需要介入。

影响：
- 巡检本身不改数据，但会直接影响后续是否需要手工干预。

可撤销性：
- 只读排查无需撤销。

### 任务 2：审批一个待办工单

路径：`ITSM > 我的待办 > 打开工单详情 > 通过 / 驳回`

步骤：
1. 在“我的待办”列表中找到目标工单。
2. 打开详情页，先检查表单数据、时间线和当前处理上下文。
3. 点击“通过”或“驳回”。
4. 填写处理意见，提交审批结果。

结果：
- 审批意见会写入流程上下文，供流程决策岗决定下一步。

影响：
- 会直接推动工单进入后续节点，或让流程回退、终止。

可撤销性：
- 审批提交后不属于简单的本页撤销动作，应按后续流程继续处理。

### 任务 3：对异常工单做人工接管

路径：`ITSM > 工单监控 > 工单详情`

步骤：
1. 打开异常工单详情，确认当前状态、责任人和时间线。
2. 需要更换责任人时执行“指派”。
3. 需要终止流程时执行“取消工单”，并填写取消原因。
4. 若是自己提交且还未结束的工单，可执行“撤回工单”。

结果：
- 工单会转给新的处理人，或进入取消、撤回后的终态。

影响：
- 这些操作会直接影响真实业务处理链路和后续审计记录。

可撤销性：
- 指派可再次调整；取消和撤回属于高影响动作，应视为谨慎操作。

## 六、风险动作与影响

- “通过 / 驳回”会改变流程决策方向，不是简单备注。
- “指派”会把责任切换给新处理人，错误指派可能导致工单继续积压。
- “取消工单”会终止当前工单生命周期，应只在业务确认后执行。
- “撤回工单”通常由提交人触发，会把单据从处理中拉回，影响上下游协作。
- 详情页展示流程图和时间线，适合先看上下文再执行动作，不建议在不了解当前步骤时直接处理。

## 七、与上下游模块的关系

- [服务目录](./service-catalog-management.md)：工单来源于服务目录里的服务定义，不同服务决定不同的流程与动作。
- `我的工单`、`我的待办`、`历史工单`：这三个入口分别对应提交者视角、审批者视角和已处理记录视角。
- `服务台`：用户通常先在服务台提交请求，再回到工单模块跟踪进度。
- `SLA 管理` 和 `优先级管理`：工单详情中展示的 SLA 状态和优先级颜色来自这些配置模块。

## 八、常见误解、限制、边界条件

- “工单监控”不是只给管理员看大盘，它同时是详情页和人工接管的主入口。
- “我的待办”与“历史工单”不是同一张列表的不同筛选，而是两个语义明确的审批入口。
- 终态工单包括完成、驳回、撤回、取消、失败；看到结束状态时，应先确认是不是正常收敛，而不一定是异常。
- 页面会自动刷新，适合观察流程推进，但自动刷新不等于审批结果立刻完全落稳，关键操作后仍建议复核时间线。
